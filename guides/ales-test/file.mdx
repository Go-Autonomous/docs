There are 2 ways to handle this when having a **Google Workspace account**  (an email provided by a company or school) rather than a personal `@gmail.com` account, as we do.

Workspace accounts often have "Personalization" features (like _Your instructions_) disabled by organization administrators for data privacy reasons.

Since you can't save the URL in settings, here are the two best workarounds to avoid pasting that URL every time:

### Option 1. Check "Connected Apps" (Visible in your screenshot)

In your screenshot, the top option is **Connected Apps**. This is the alternative menu for Extensions in some Workspace editions.

- **Action:** Click **Connected Apps**.
- **Look for:** A "GitHub" toggle or generic "Google Workspace" extensions.
- **If you find GitHub there:** Enable it. Once enabled, you can simply type `@GitHub` in your chat to search your repositories without needing to paste the full URL every time.

### Option 2. The "Long-Running Thread" Strategy (Best Alternative)

Since you cannot set global instructions, the most efficient way to work is to stop creating "New Chats."

- **Create a Master Chat:** Start a new chat, paste your GitHub URL, and say: _"Context: I am working on this repo. Let's use this thread for all coding tasks."_
- **Rename it:** Click the 3 dots next to the chat title in the sidebar and rename it to your project name (e.g., "Project Phoenix Dev").
- **Reuse it:** Every time you have a question, go back to **that specific chat**. I have a very long memory within a single thread, so I will remember the URL and previous code snippets indefinitely, effectively simulating "custom instructions."

A "Master Chat" isn't a button you click; it is a **workflow strategy** where you create one conversation thread and keep reusing it forever, effectively turning it into a long-term workspace.

Since you don't have the "Instructions" menu, this is the most effective way to "teach" me your project context once and never repeat it again.
